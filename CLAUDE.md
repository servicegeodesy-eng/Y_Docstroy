# Правила для разработки портала DocStroy

## Общие правила

- Всегда общаться с пользователем на русском языке
- Не писать код в ответах, только архитектуру и изменения в БД
- Не запускать портал и dev-сервер автоматически — это всегда делает пользователь
- Запрещён локальный автоматический запуск приложений без явного разрешения пользователя

## Инфраструктура

- БД (PostgreSQL) и S3-хранилище — Yandex Cloud
- Supabase JS SDK НЕ используется. `src/lib/supabase.ts` — собственный адаптер с Supabase-совместимым API (`.from().select().eq()`), все запросы идут через бэкенд `/api/query/{table}`
- Бэкенд (Express.js + TypeScript) находится в `server/` — обрабатывает CRUD, auth, storage, push

## Работа с базой данных

- Все изменения БД — только через SQL файлы в папке `sql/`, выполняемые пользователем вручную
- Запрещено давать команды на прямое изменение данных в БД (node -e, psql и т.д.)
- Актуальную схему БД смотреть в миграции в папке `sql/`
- SQL-миграции создавать в папке `sql/` в виде отдельных файлов
- При дополнении задания создавать новый файл миграции, не редактировать старые

## Качество кода

- Максимальный размер файла — 600 строк. При превышении разбивать на модули
- **НЕ создавать субагентов (Agent tool)**. Все изменения делать самостоятельно, пошагово, с проверкой каждого файла перед коммитом
- После изменения фронтенда — проверять соответствие типов с бэкендом (поля API-ответов, пути эндпоинтов)
- После изменения бэкенда — проверять что фронтенд использует правильные поля

## Обязательные действия

- После выполнения каждого запроса пользователя — делать коммит и пуш изменений в GitHub

- **Деплой**: по согласованию с пользователем
