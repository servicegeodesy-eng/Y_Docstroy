# Правила для разработки портала DocStroy

## Общие правила

- Всегда общаться с пользователем на русском языке
- Не писать код в ответах, только архитектуру и изменения в БД
- Не запускать портал и dev-сервер автоматически — это всегда делает пользователь
- Запрещён локальный автоматический запуск приложений без явного разрешения пользователя

## Работа с базой данных

- Актуальную схему БД смотреть в миграции в папке `sql/`
- SQL-миграции создавать в папке `sql/` в виде отдельных файлов
- При дополнении задания создавать новый файл миграции, не редактировать старые

## Качество кода

- Максимальный размер файла — 600 строк. При превышении разбивать на модули

## Обязательные действия

- После выполнения каждого запроса пользователя — делать коммит и пуш изменений в GitHub

- **Деплой**: по согласованию с пользователем
